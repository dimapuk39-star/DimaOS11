# Публикация DimaOS 11 на GitHub Pages

## 1. Создайте репозиторий

На GitHub нажмите **New repository**, задайте имя, например `DimaOS11`, и создайте публичный репозиторий.

## 2. Загрузите проект

Загрузите **содержимое папки DimaOS11** в корень репозитория. В корне GitHub должны находиться `package.json`, `index.html`, папки `src` и `.github`.

Не загружайте папки `node_modules` и `dist`: GitHub Actions создаст их самостоятельно.

## 3. Включите GitHub Pages

Откройте репозиторий и перейдите:

`Settings → Pages → Build and deployment → Source → GitHub Actions`

## 4. Дождитесь публикации

После загрузки файлов откройте вкладку **Actions**. Задача `Deploy DimaOS 11 to GitHub Pages` соберёт и опубликует сайт.

Адрес будет выглядеть так:

`https://ВАШ-ЛОГИН.github.io/DimaOS11/`

Каждый следующий push в ветку `main` или `master` автоматически обновит сайт.

## Ограничения статической версии

- Приложения, окна, обои, фотографии, плеер, Paint и настройки работают в браузере и сохраняются локально.
- Характеристики определяются доступными браузерными API. Точная модель процессора и диски недоступны веб-сайту из-за защиты браузера.
- Локальные команды Dima AI работают. Для OpenAI нужен отдельный защищённый сервер: API-ключ нельзя публиковать в коде GitHub Pages.
- `START-DimaOS.cmd` можно оставить для локального режима с точными характеристиками и серверным Dima AI.
